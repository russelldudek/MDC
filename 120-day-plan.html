120-Day Entry Plan

Instrument the company. Stabilize the load path. Release founder attention.

Read before redesign

Learn where operating truth exists and where the team has stopped trusting it.

One record before more meetings

Make the engagement the unit connecting sales, delivery, finance, staffing, agents, proof, and renewal.

Thresholds before heroics

Define when work proceeds, changes, escalates, pauses, or returns to the founder.

0-30 days

Read the system and establish the baseline

Listen

  • Founder dependency interviews
  • Client and delivery-lead listening
  • Sales handoff observation
  • Finance and cash-flow review
  • Agent owner / user interviews

Map

  • Active engagement portfolio
  • Promise-to-proof chain
  • Pipeline-to-capacity logic
  • Escalation and decision rights
  • Data conflicts and blind spots

Instrument

  • Weekly scorecard baseline
  • Engagement economics view
  • Founder attention log
  • Agent job and evaluation register
  • Client outcome evidence inventory
31-60 days

Close the operating gaps

  • Install the Engagement Load Record: one shared source for promise, scope, capacity, economics, risk, evidence, renewal, and owner.
  • Define sales-to-delivery quality control: pre-sign review triggers for novelty, margin sensitivity, agent dependence, client readiness, and staffing risk.
  • Set scope-change mechanics: observable triggers, client conversation language, commercial authority, and margin impact.
  • Reconcile forecast and actuals: pipeline stage, signed work, staffing, utilization, revenue, margin, billing, and cash.
  • Publish the decision-rights charter: ownership boundaries, standing sync, escalation thresholds, attention budget, and review cadence.
  • Publish metric definitions: eliminate competing interpretations of utilization, realization, backlog, margin, renewal, and capacity.
Day-60 gate

The weekly operating review produces decisions without reconstructing the business from multiple spreadsheets and private explanations.

61-90 days

Build capacity ahead of demand

  • Pipeline-weighted capacity plan: demand confidence, skills, seniority mix, utilization floor, hiring lead time, and contingency coverage.
  • Remote onboarding sprint: artifact-based path to client context, method, tools, quality, operating cadence, and escalation.
  • Manager operating standard: coaching, engagement reviews, escalation handling, performance signals, and written async habits.
  • Agent management standard: job definition, owner, evaluation set, human authority, escalation, cost threshold, release posture, and quarterly review.
  • Renewal evidence ritual: begin outcome documentation before closeout, not after the client asks what changed.
Day-90 gate

New work can be staffed, started, reviewed, and escalated through a repeatable system without founder interpretation at every handoff.

91-120 days

Run the company rhythm beneath the founder

  • Founder leverage test: run delivery, sales handoff, staffing, and escalation without founder attendance; return only named exceptions.
  • Client evidence test: show accepted outcomes, adoption, capability transfer, or financial/operating evidence before renewal.
  • Economics test: compare quoted assumptions to actual staffing, utilization, change burden, agent leverage, margin, billing, and cash.
  • Forecast test: locate error in pipeline quality, handoff, capacity, delivery, billing, or collections and update the model.
  • Agent redesign test: select one workflow using value, repeatability, data readiness, human authority, evaluation feasibility, and owner commitment.
Day-120 gate

The founder can remain primarily external while operating decisions, client evidence, economics, and exceptions remain visible and trusted.

Clients

Outcome evidence begins before closeout and informs renewal.

Economics

Quoted assumptions reconcile to utilization, margin, billing, and cash.

Forecast

Variance produces a corrected model and a named owner.

Bench

Demand can be covered without blind idle cost or emergency hiring.

Agents

Each system has a job, owner, evaluation, authority, cost, and release posture.

Founder leverage

Routine dependencies shrink without losing market signal or decision quality.

What should be true by day 120

  • Client work has one visible path from promise through evidence and renewal.
  • Delivery, sales, staffing, finance, and agent owners use shared definitions.
  • Forecast variance produces learning rather than explanation theater.
  • The next AI-enabled workflow is selected because economics and operating conditions support it.
